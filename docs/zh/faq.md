# 常见问题

**我的数据会去哪里？**
除了在你的 RootPilot 容器、你自己的主机（通过 SSH）和你用自己密钥指定的 LLM API 之间流动之外，哪里都不去。没有 RootPilot 托管的中间人，也没有遥测。参见 [security.md](security.md)。

**支持哪些 LLM 模型？**
自带密钥。RootPilot 可与主流 LLM API 配合使用；由你选择提供商和模型，费用记在你自己的账户上，而非我们的。

**RootPilot 会改动或搞坏我的服务器吗？**
不会。它只运行一组固定的只读检查命令 —— 没有任意命令执行功能，哪怕在确认之后也没有。完整的 38 条命令白名单见 [security.md](security.md)。

**需要在每台服务器上装 agent 吗？**
不需要。目标服务器只需接受你的 SSH 连接。RootPilot 本身是你唯一要安装的东西，并且只在一台机器上运行。

**它和 Prometheus / Grafana / Datadog 有什么区别？**
监控告诉你某个指标越过了阈值。RootPilot 从这里接手，进一步解释*为什么* —— 它从受影响的主机采集证据，产出一份根因报告。它可以读取你的 Prometheus 来获取趋势上下文，但它是对监控的补充，而非替代。

**控制台上有 2FA / 登录吗？**
控制台仅绑定 `127.0.0.1`，其设计是通过 SSH 隧道，或放在你自己带认证的反向代理之后来访问 —— 这就是访问控制的边界。不要把 18081 端口直接暴露到公网。

**它到底有多准？**
在当前的校准集上：**在 29 个标准故障场景中达到 89.7% 的根因准确率，且对健康主机零误报** —— 使用 38 条只读命令。准确率取决于你自带的模型以及主机暴露出多少证据。我们公开方法论，并持续扩充场景库。

**如何导入 license？**
在控制台的 license 页面粘贴 license 字符串。免费版无需 license；Pro/Max 的 license 来自 [rootpilotx.com](https://rootpilotx.com)。license 在本地验证 —— 运行时无需调用 license 服务器。

**资源要求是什么？**
Docker 20.10+、Docker Compose v2，以及供应用 + MySQL 使用的约 2 GB 空闲内存。它可以在一台小型 VM 上流畅运行。

**出了故障但 RootPilot 判断错了，怎么办？**
请提一个 issue，附上（脱敏后的）证据以及真实原因最终是什么。真正的漏判会成为校准案例 —— 准确率数字正是这样提升的。
